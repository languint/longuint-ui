DockableFrames are a variant of the [Frame](/reference/frame/) component.
DockableFrames automatically resize and shift their position to overlapping regions.

### Usage

<Aside>DockableFrames without defined regions are still draggable.</Aside>

```tsx
export const regions: DockableRegion[] = [
	{ size: new UDim2(0.25, 0, 1, 0), position: new UDim2(0, 0, 0, 0) }, // Left Panel
	{ size: new UDim2(0.25, 0, 1, 0), position: new UDim2(0.75, 0, 0, 0) }, // Right Panel
	{ size: new UDim2(0.5, 0, 0.25, 0), position: new UDim2(0.25, 0, 0, 0) }, // Top Panel
	{ size: new UDim2(0.5, 0, 0.25, 0), position: new UDim2(0.25, 0, 0.75, 0) }, // Bottom Panel
];

<DockableFrameComponent dockableRegions={regions}></DockableFrameComponent>
```

DockableFrames, if used incorrectly are prone to [Dodging](/guides/dockableframes/dodging/) their intended target.

It is recommended to use percentage based sizing of regions to avoid situations where regions collide on different screen sizes.
